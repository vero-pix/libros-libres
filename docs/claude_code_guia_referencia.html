3. Configuracion

Archivos de configuracion (por prioridad)

AlcanceUbicacionCompartido?
Managed (IT)Sistema / /etc/claude-code/Si
Local proyecto.claude/settings.local.jsonNo (gitignored)
Proyecto.claude/settings.jsonSi (git)
Usuario~/.claude/settings.jsonNo

Ejemplo de settings.json

{
  "model": "claude-opus-4",
  "effort": "medium",
  "theme": "dark",
  "defaultMode": "default",
  "permissions": {
    "allow": ["Bash(npm *)", "Read"],
    "ask": ["Bash(rm *)"],
    "deny": ["Bash(sudo *)"]
  },
  "autoMemoryEnabled": true,
  "env": {
    "CUSTOM_VAR": "value"
  }
}

Opciones principales

SettingValoresDescripcion
modelclaude-opus-4, claude-sonnet, etc.Modelo de IA
effortlow, medium, high, max, autoNivel de esfuerzo
themedark, light, colorblind-daltonized, ansiTema visual
defaultModedefault, acceptEdits, plan, auto, etc.Modo de permisos inicial
autoMemoryEnabledtrue/falseMemoria automatica
additionalDirectoriesarray de rutasDirs extra accesibles

4. Sistema de Permisos

Modos de permiso

ModoComportamientoCuando usarlo
defaultPide permiso en primer usoTrabajo general seguro
acceptEditsAuto-acepta ediciones de archivosIteracion rapida
planSolo lectura, no modifica nadaRevision de codigo
autoAuto-aprueba con clasificador de seguridadEntornos confiables
dontAskDeniega todo excepto lo pre-aprobadoEntornos restringidos
bypassPermissionsSalta prompts (excepto dirs protegidos)Solo VMs aisladas
Tip: Usa Shift+Tab para cambiar rapidamente entre modos durante la sesion.

Sintaxis de reglas

"permissions": {
  "allow": [
    "Bash(npm run *)",        // Cualquier npm run
    "Read",                   // Toda lectura
    "WebFetch(domain:github.com)"  // Solo github
  ],
  "deny": [
    "Bash(sudo *)",           // Bloquear sudo
    "Edit(.env)"              // No editar .env
  ]
}

Directorios protegidos (siempre piden permiso)

  • .git/
  • .claude/ (excepto commands/, agents/, skills/)
  • .vscode/, .idea/, .husky/

5. Hooks (Automatizacion)

Los hooks ejecutan acciones automaticas en respuesta a eventos.

Tipos de hook

TipoComo funciona
commandEjecuta script shell
httpEnvia POST a webhook
promptEvaluacion con LLM (si/no)
agentSubagente para verificacion compleja

Eventos disponibles

EventoBloquea?Cuando se dispara
PreToolUseSiAntes de ejecutar herramienta
PostToolUseNoDespues de ejecutar herramienta
UserPromptSubmitSiAl enviar mensaje
SessionStartNoAl iniciar sesion
NotificationNoAlertas
FileChangedNoCambio detectado en archivo
StopSiAntes de detenerse

Ejemplo: bloquear comandos destructivos

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "/ruta/a/validar.sh"
      }]
    }]
  }
}

6. Sistema de Memoria

CLAUDE.md — Instrucciones del proyecto

ArchivoAlcanceCompartido?
./CLAUDE.mdEquipo, este repoSi (git)
./CLAUDE.local.mdTu, solo este repoNo
~/.claude/CLAUDE.mdTu, todos los proyectosNo
.claude/rules/*.mdReglas por temaSi (git)
Tip: Manten CLAUDE.md bajo 200 lineas. Usa encabezados y bullets. Se especifica: "Usar 2 espacios de indentacion" en vez de "Formatear bien el codigo".

Importar archivos en CLAUDE.md

Ver @README para resumen y @package.json para scripts.
Reglas adicionales: @docs/guia-git.md

Auto Memory — Notas entre sesiones

Claude guarda notas automaticamente en ~/.claude/projects/<proyecto>/memory/

  • MEMORY.md — Indice (se carga al inicio, max 200 lineas)
  • Archivos por tema — Se cargan bajo demanda
  • Edita con /memory o directamente los archivos

Reglas con scope (paths)

---
paths:
  - "src/api/**/*.ts"
---

# Reglas API
- Validar input en todos los endpoints
- Usar formato de error estandar

10. MCP (Model Context Protocol)

MCP conecta herramientas externas a Claude Code.

Configurar servidor MCP

En ~/.claude/mcp.json o .claude/mcp.json:

{
  "mcpServers": {
    "mi-servidor": {
      "command": "npx",
      "args": ["@ejemplo/mcp-server"],
      "env": {
        "API_KEY": "${MI_API_KEY}"
      }
    }
  }
}

Gestionar

Usa /mcp para agregar, eliminar, testear servidores OAuth.

Permisos MCP

"allow": [
  "mcp__servidor",                 // Todas las herramientas
  "mcp__servidor__herramienta"    // Herramienta especifica
]

11. Integraciones IDE

VS Code

  • Instalar extension desde marketplace
  • Cmd+Shift+L — Compartir seleccion con Claude
  • Cambios se muestran en diff de VS Code
  • Resaltado de sintaxis inline

JetBrains (IntelliJ, WebStorm, etc.)

  • Instalar plugin "Claude Code" desde marketplace
  • Cmd+Esc (Mac) / Ctrl+Esc (Windows) — Abrir Claude
  • Cmd+Option+K — Insertar referencias a archivos
  • Diagnosticos automaticamente compartidos

Usa /ide para verificar estado de conexion.

12. Funciones Avanzadas

Pensamiento extendido

Activa con Cmd+T o /config. Claude "piensa" mas profundo antes de responder.

Subagentes

Claude puede lanzar agentes especializados en paralelo. Crea en .claude/agents/:

---
description: Agente especializado en testing
model: claude-opus-4
---

Instrucciones del agente...

Batch (cambios masivos)

/batch migrar todos los componentes a TypeScript — Lanza 5-30 agentes en worktrees paralelos.

Dictado por voz

/voice — Push-to-talk. Requiere cuenta claude.ai.

14. Solucion de Problemas

ProblemaSolucion
Claude no sigue CLAUDE.md/memory para verificar. Hacer instrucciones mas especificas.
Auto-memory no funcionaVerificar autoMemoryEnabled: true. Revisar con /memory.
Permisos piden mucho/permissions para agregar allow rules.
Skill no se activaHacer description mas especifica. Invocar con /nombre.
IDE no conectaReiniciar IDE. Verificar plugin. Ejecutar /ide.
Contexto lleno/compact. Revisar que CLAUDE.md tenga < 200 lineas.
Plugin no cargaVerificar plugin.json. Ejecutar /reload-plugins.
